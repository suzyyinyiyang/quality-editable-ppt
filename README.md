# Quality Editable PPT Skill

让高视觉 PPT 真正可编辑的 Codex Skill。

这个 Skill 的目标不是简单生成一份 PPT，而是帮助用户按照标准流程：**先确认内容方向，再确认视觉预览，最后生成高还原、可编辑的 PowerPoint 文件**。

## 解决什么问题

普通 AI 生成 PPT 时，常见问题是：

- 预览效果不错，但导出的 PPT 视觉还原差。
- 页面只是图片，后续不能编辑文字、图表、表格。
- 图标、卡片、间距、文字位置、背景效果容易跑偏。
- 复杂页面需要大量人工二次调整。

`Quality Editable PPT` 通过“预览图锁定 + 图层拆解 + 可编辑重构”的方式，尽量在保证视觉效果的同时，让标题、正文、图表、表格、流程、卡片等关键内容保持可编辑。

## 核心能力

- **高视觉预览确认**：先生成高视觉 PPT 预览图，确认后再进入 PPT 生成。
- **高还原复刻**：锁定预览图作为唯一视觉参考，避免后续重新设计导致偏差。
- **内容可编辑**：标题、正文、数字、流程、表格、标准图表等尽量保持可编辑。
- **复杂视觉资产化**：复杂背景、主视觉、纹理、光效、复杂 icon 可作为图片资产处理，优先保证视觉还原。
- **图表和表格原生化**：标准图表优先使用 PPT 原生图表，真实数据表优先使用 PPT 原生表格。

## 适用场景

1. **只有主题，从 0 开始生成 PPT**  
   适合论坛演讲、产品介绍、方案汇报、发布会材料等。

2. **已有长文本或大纲，生成高视觉 PPT**  
   适合把方案文档、汇报材料、产品说明整理成正式 PPT。

3. **已有 PPT 图片，复刻成可编辑 PPT**  
   适合将确认好的预览图、参考图或旧版图片稿，还原为可编辑 PowerPoint。

## 基本工作流

```text
输入主题 / 大纲 / 参考图
→ 确认大纲方向
→ 确认页面结构与视觉策略
→ 生成高视觉预览图
→ 锁定预览图
→ 图层拆解
→ 生成可编辑 PPT
→ 前 1–3 页渲染对照确认
→ 输出整套 PPT
```

## 安装方式

将整个 `quality-editable-ppt` 文件夹复制到 Codex Skill 目录下。

### macOS

```bash
~/.codex/skills/quality-editable-ppt/
```

最终结构应为：

```text
~/.codex/skills/quality-editable-ppt/SKILL.md
```

### Windows

```text
C:\Users\你的用户名\.codex\skills\quality-editable-ppt\
```

最终结构应为：

```text
C:\Users\你的用户名\.codex\skills\quality-editable-ppt\SKILL.md
```

## 使用示例

### 从主题生成 PPT

```text
$Quality Editable PPT，使用这个 skill，做 10 页 16:9 可编辑 PPT。
主题：《终端安全 AI 智能运营》
风格：蓝绿产品发布风
请先生成大纲给我确认，确认后再生成视觉预览图，预览图确认后再生成可编辑 PPT。
```

### 从参考图还原 PPT

```text
$Quality Editable PPT，使用这个 skill，把我上传的 PPT 图片还原为 16:9 可编辑 PPT。
请锁定当前图片作为唯一视觉参考，完成图层拆解后生成可编辑 PPT，并先给我前 1–3 页对照确认。
```
